Developer documentation

Run Resonara in lite desktop mode, stand up the full Docker stack, package installers, and exercise the APIs.

Requirements

  • Node.js 20+
  • npm
  • ffmpeg & ffprobe on PATH
  • Docker Compose (full stack only)

Desktop lite (recommended)

# clone & install
git clone https://github.com/Yuri-Lima/resonara.git
cd resonara
npm install
npm run build
npm run desktop:dev    # Electron + lite API :3847

# API only
RESONARA_LITE=1 PORT=3000 npm run start:lite

Open /ui/, /ui/piano/, /ui/voice/, and Swagger at /docs.

Full stack

docker compose up -d postgres redis minio minio-init
npm run start:dev      # API :3000 without RESONARA_LITE

macOS DMG Windows Setup All releases

Latest release: v2.1.0

Packaging

npm run dist:mac    # release/Resonara-2.1.0-arm64.dmg , *.zip
npm run dist:win    # release/Resonara Setup 2.1.0.exe (Windows / CI)
npm run dist:all

Config lives in package.json → build. Artifacts are written to release/. Version is package.json 2.1.0.

Key APIs

  • GET /health — product, mode, ffmpeg, TTS
  • POST /tts/synthesize — long-form system TTS job
  • POST /tracks/upload · normalize · transcode · trim
  • GET /piano/packs · takes · export
  • WebSocket /jobs — subscribe → job:progress

Tests

npm test
npm run smoke:tts
npm run smoke:service

v2.1 limits

  • Unsigned installers by default
  • ffmpeg not bundled — install on the host
  • Expressive (Chatterbox) pack is optional and multi-GB — not in base DMG/NSIS
  • No Linux installer yet
